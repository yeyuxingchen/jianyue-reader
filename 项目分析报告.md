## 四、性能分析

### 4.1 内存风险点

| 风险点 | 位置 | 严重度 | 影响 | 优化方案 |
|--------|------|--------|------|----------|
| EPUB 全量解压 | `main.ts:472-490` | 🔴 高 | 大文件 (>100MB) 可能导致内存溢出 | 按需加载 |
| 书籍列表全量加载 | `stores/library.ts` | 🟡 中 | 书籍数量多时内存占用高 | 分页加载 |
| TTS 音频队列 | `services/ttsPlayer.ts` | 🟡 中 | 长时间朗读内存增长 | 限制队列大小 |
| 封面图片 Blob URL | `utils/cover.ts` | 🟢 低 | 已有清理机制 | - |

#### EPUB 解压问题详解

```typescript
// 当前实现: 全量解压并通过 IPC 传输 (main.ts:472-490)
ipcMain.handle('epub:extractAll', (_event, filePath: string) => {
  const zip = new AdmZip(filePath)
  const entries = zip.getEntries()
  const result: { name: string; data: number[] }[] = []
  
  for (const entry of entries) {
    if (entry.isDirectory) continue
    const data = Array.from(entry.getData())  // Buffer → number[] 昂贵转换
    result.push({ name: entry.entryName, data })
  }
  return result  // 全量传输 ⚠️
})
```

**问题**:
1. 整个 EPUB 解压到内存
2. Buffer 转 number[] 开销大
3. IPC 序列化/反序列化开销
4. 渲染进程内存占用翻倍

### 4.2 CPU 热点

| 热点 | 位置 | 频率 | 优化建议 |
|------|------|------|----------|
| JSON 序列化 | `dbService.ts` | 每次数据变更 | 使用 SQLite |
| 轮询等待 | 多处 | 100ms 间隔 | 改用事件/回调 |
| 全文搜索 | `SearchPanel.vue` | 用户触发 | 添加索引 |
| 列表渲染 | `Bookshelf.vue` | 滚动时 | 虚拟滚动 |

### 4.3 轮询机制统计

| 位置 | 间隔 | 最大等待 | 问题 | 优化方案 |
|------|------|----------|------|----------|
| `NoteEditor.vue:212-228` | 100ms | 无限 | 编辑器初始化 | 事件监听 |
| `ttsPlayer.ts:114-128` | 100ms | 30s | 音频等待 | Promise + 回调 |
| `main.ts:702-730` | 100ms | 5s | 浮动窗口数据注入 | did-finish-load |
| `main.ts:823-851` | 100ms | 5s | 浮动窗口数据注入 | did-finish-load |

**总计**: 4处轮询机制，建议全部改为事件驱动。

### 4.4 存储性能

#### 当前方案
```typescript
// electron-store: JSON 文件存储
function updateBook(bookId: string, updates: Partial<Book>) {
  const idx = books.value.findIndex((b) => b.id === bookId)
  if (idx !== -1) {
    books.value[idx] = { ...books.value[idx], ...updates }
    db.saveBooks(books.value)  // 每次都序列化全部书籍 ⚠️
  }
}
```

**问题**:
1. **全量序列化**: 每次更新都序列化整个数组
2. **无索引**: 搜索需要全量扫描
3. **无分页**: 无法支持大量数据

### 5.4 IPC 安全分析

```typescript
// 当前: 直接使用用户输入的路径 (main.ts:258-274)
ipcMain.handle('fs:readFileAsBuffer', (_event, filePath: string) => {
  const buffer = fs.readFileSync(filePath)  // 无路径验证 ⚠️
```
