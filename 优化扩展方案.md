### 3.1 CSP 配置优化 [P0]

**目标**: 消除 XSS 攻击面

**当前问题**:
```html
<meta http-equiv="Content-Security-Policy" content="
  default-src 'self' 'unsafe-inline' 'unsafe-eval' blob: data: file: cacheimg:;
  connect-src 'self' https: http: ws: wss: blob: data:;
" />
```

**优化方案**:
```html
<meta http-equiv="Content-Security-Policy" content="
  default-src 'self' 'unsafe-inline' blob: data: file: cacheimg:;
  script-src 'self' 'unsafe-inline';
  style-src 'self' 'unsafe-inline';
  connect-src 'self' https: http: blob: data:;
```

### 3.3 文件路径验证 [P0]

**目标**: 防止任意文件读取

**实现方案**:
```typescript
// electron/security.ts
import { app } from 'electron'
import path from 'path'

const ALLOWED_DIRS = [
  app.getPath('userData'),
  app.getPath('documents'),
  app.getPath('downloads'),
]

export function isPathAllowed(filePath: string): boolean {
  const resolvedPath = path.resolve(filePath)
  return ALLOWED_DIRS.some(dir => resolvedPath.startsWith(dir))
}
```

### 3.4 敏感数据加密 [P0]

**目标**: 保护 API Key 等敏感信息

**实现方案**:
```typescript
import { safeStorage } from 'electron'

export function encryptData(data: string): string {
  if (safeStorage.isEncryptionAvailable()) {
    const encrypted = safeStorage.encryptString(data)
    return encrypted.toString('base64')
  }
  return data
}

export function decryptData(encryptedData: string): string {
  if (safeStorage.isEncryptionAvailable()) {
    const buffer = Buffer.from(encryptedData, 'base64')
    return safeStorage.decryptString(buffer).toString()
  }
  return encryptedData
}
```

**实施步骤**:

| Vue Router 重构引入 bug | 中 | 中 | 逐步迁移，保留旧代码 | 恢复旧页面切换逻辑 |
| 性能优化效果不明显 | 低 | 中 | 基准测试，量化对比 | 调整优化策略 |

### 8.2 进度风险
